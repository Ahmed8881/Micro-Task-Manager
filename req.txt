  - priority ENUM('Low','Medium','High') DEFAULT 'Medium'

--- API (PHP) ---
Base URL: /api/

General API rules:
- Use JSON request bodies where applicable (POST/PUT).
- Use prepared statements for all DB queries.
- Return JSON `{ "status": "success"|"error", "data": ..., "message": "...", "code": HTTP_CODE }`.
- Implement proper CORS headers and method handling (OPTIONS, GET, POST, PUT, DELETE).
- Validate inputs and return 400 on bad requests, 404 for not found.

Endpoints:

1) Categories
- GET /api/categories
  -> list categories
- POST /api/categories
  Body: { "name": "New Category" }
- DELETE /api/categories/{id}

2) Tasks
- GET /api/tasks
  Query params: status, category_id, search, priority, assigned_to, sort (due_date|priority|created_at), page, per_page
  -> supports pagination and returns `{ pagination: { total, page, per_page } }`
- GET /api/tasks/{id}
  -> returns task with subtasks and comments
- POST /api/tasks
  Body: { title, description, priority, due_date, status, category_id, assigned_to, subtasks: [{title}, ...] }
  -> create task + subtasks, add activity log
- PUT /api/tasks/{id}
  Body: partial or full task fields, also allow subtasks update (create/update/delete)
  -> update and return updated task, log activity
- DELETE /api/tasks/{id}
  -> delete task, subtasks, comments, log activity
- POST /api/tasks/{id}/move
  Body: { status: 'in_progress' | 'done' | 'todo' }
  -> change status, log activity

3) Subtasks
- PUT /api/subtasks/{id}  -> toggle is_done or update title
- DELETE /api/subtasks/{id}

4) Comments
- POST /api/tasks/{id}/comments
  Body: { author, content }
- GET /api/tasks/{id}/comments

5) Activity log
- GET /api/tasks/{id}/activity

6) Export
- GET /api/tasks/export?format=csv -> returns CSV of tasks

7) Real-time updates (optional)
- GET /api/stream/updates  (SSE) -> sends simple events when tasks changed. Include a note: if server SSE not supported on target environment, front-end should fallback to polling every 5 seconds.
